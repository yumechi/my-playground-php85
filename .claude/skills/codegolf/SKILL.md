# PHP コードゴルフモード

このスキルが有効なとき、コードの短さを最優先にする。可読性・安全性・保守性は無視してよい。

## スコアルール

- スコア = コード中の全 ASCII 空白文字（スペース、タブ、改行等）を除去した後のバイト数
- 先頭や末尾に置かれた PHP タグ (`<?php`、`<?`、`?>`) はカウントしない
- 1バイトでも短い方が良い

## 実行環境

- PHP 8.5.3-cli
- `error_reporting` は `E_ALL & ~E_WARNING & ~E_NOTICE & ~E_DEPRECATED` に設定済み（Warning/Notice/Deprecated は無視してよい）
- `make run` で実行、`make run FILE=other.php` で別ファイル指定
- 標準入力: `echo "入力" | make run`

## ジャッジ環境の制約

- mbstring 以外の拡張は無効
- ファイルアクセス・ネットワークアクセス不可
- 外部コマンド実行不可
- `readline()` は動かない可能性がある — `fgets(STDIN)` を使う

## 最適化テクニック

以下のパターンを積極的に適用する:

- **数値リテラル**: `1000` → `1e3`、`100` → `1e2` で短縮
- **カウントダウンループ**: `for($i=N;$i--;)` は `$i++` が不要で短い
- **累積計算**: 「定数から減算」パターンで出力時の式を短くできる
- **リテラル改行**: echo 内の実際の改行文字はスコア 0 バイト（空白除去されるため）
- **for 式内の改行**: for の式内に print/文字列リテラルで改行を含めるとパースエラーになる（PHP 8.5.3 で確認済み）
- **変数の初期化省略**: 未初期化変数は Warning だが抑制済みなので利用可能
- **短い関数名**: `join` vs `implode` など、同機能で短い方を選ぶ
- **三項演算子/null合体**: if/else より短い場合が多い
- **文字列のインデックスアクセス**: `$s[$i]` は配列化不要で直接使える

## 出力フォーマット

コードを提示するとき:
1. コードそのもの
2. スコア（空白除去・PHPタグ除外後のバイト数）

スコアの計算は正確に行うこと。
